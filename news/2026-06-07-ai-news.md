### 3. Microsoft、Build 2026でMAIモデル7本を発表——OpenAI依存からの脱却を加速

Microsoft Build 2026（6月2日）でMicrosoftが発表した**MAIモデルファミリー**の全容が明らかになった。7本すべてがOpenAIのデータを使わずゼロから自社学習させた「完全内製モデル」で、Azure AI Foundryで提供される。

| モデル名 | 特徴 |
|---|---|
| **MAI-Thinking-1** | 推論フラッグシップ、35Bアクティブパラメーター、256Kコンテキスト |
| **MAI-Code-1-Flash** | コーディング特化、5Bアクティブパラメーター、GitHub Copilot統合 |
| **MAI-Image-2.5** | テキスト→画像／画像→画像、Arena AIリーダーボード3位 |
| **MAI-Transcribe-1.5** | 音声認識（STT） |
| **MAI-Voice-2** | 音声合成（TTS） |

MAI-Thinking-1は「OpenAI依存脱却の礎」として業界に注目されており、Fireworks AI・Baseten・OpenRouterなどAzure外のプラットフォームでも利用可能になっている。

### 4. 世界初・LLMエージェントによる自律サイバー攻撃をSysdigが記録——113秒でDB全件窃取

セキュリティ企業Sysdigの脅威リサーチチームが、**史上初めて実環境でLLMエージェントが自律的に動作したサイバー攻撃インシデント**を詳細記録し、公開した。

攻撃の流れは以下の4ステップ（4ピボット）で構成された：

1. **CVE-2026-39987**（Marimo ノートブック RCE）で初期侵入
2. 侵害ホストからAWSクラウド認証情報を2件抽出
3. AWS Secrets ManagerからSSH秘密鍵を取得
4. SSHバスティオン経由で内部PostgreSQLデータベースのスキーマ全件＋データを**113秒以内に外部流出**

エージェントは各コマンドの結果を受け取り、次のアクションをリアルタイムで判断・適応しており、「スクリプトでも人手でも説明できない4種の署名が113秒以内に同時発生した」とSysdigは指摘。LLMエージェントが侵入後の攻撃フェーズを完全自動化した初の公式記録であり、従来型の防御策の有効性を根本から問い直す事例として注目されている。

- [AI agent at the wheel（Sysdig Blog）](https://www.sysdig.com/blog/ai-agent-at-the-wheel-how-an-attacker-used-llms-to-move-from-a-cve-to-an-internal-database-in-4-pivots)
- [AI Agent Conducted a Cyberattack on Its Own（Security Magazine）](https://www.securitymagazine.com/articles/102325-ai-agent-conducted-a-cyberattack-on-its-own-it-took-less-than-one-hour)
